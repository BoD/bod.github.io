>>BoD2000 seal of quality_

Je me suis imposé une charte de qualité pour ce que seront les logiciels que je ferai dorénavant. Cela s'appelle le "BoD 2000 seal of quality". Les logiciels conformes à ce label devront être :
  • en Java;
  • "multiskin" (ou "customizable", ou "skinable");
  • open source.
Le logo suivant sera mentionné pour les logiciels conformes a cette norme :




Une nouvelle version de cette norme existe : le "BoD2001 seal of quality". Cette nouvelle norme apporte quelques points tout en enlevant une restriction du BoD 2000. Les logiciels BoD2001 devront êrte :
  • "auto-updatable" (s'il existe une version plus récente du logiciel, celui-ci devra pouvoir être capable de la charger automatiquement);
  • "multiskin" (ou "customizable", ou "skinable") si applicable;
  • "language-packable" (la possibilité de changer la langue du logiciel);
  • bien que cela ne soit pas quantifiable, le logiciel devra être "customizable" au plus haut degrée, eventuellement modulaire et/ou scriptable;
  • "open source" (cela ne signifie pas forcément libre de droit);
  • dans la mesure du possible, les formats et/ou technologies utilisées devront être ouvertes (par exemple : utilisation du XML), afin que d'autres logiciels puisse s'interfacer facilement.

>>MSLQJAFQSA BoD Remind'98 v1.25_

Premier de la collection MSLQJAFQSA (Mon Super Logiciel Que J'Ai Fait Qui S'Appelle), MSLQJAFQSA BoD Remind'98 v1.25 est un utilitaire heuu.. utile. Il vous rappelle à l'avance les dates importantes comme les rendez-vous ou anniversaires, ainsi que les trucs qu'il faut pas oublier de faire ("To do"). Son utilisation est très simple : il suffit de l'ajouter dans le menu démarrage, ainsi il se lancera automatiquement au démarrage de windows, vous indiquant quotidiennement les tâches du jour.



Dates

Vous pouvez ajouter des événements ponctuels (par exemple un rendez-vous) ou répétitifs (par exemple un anniversaire). Il est aussi possible d'ajouter des événements hebdomadaires (par exemple tous les lundis...).

To do

Ils servent à se souvenir de quelque-chose à faire, sans date particulière. Une fois la tâche accomplie, il suffit de cocher la case associée, et Remind ne vous la rappellera plus.

Configuration requise

Windows 95, 98 ou NT4.

>>BoD Colored v1.20_

BoD Colored est un script pour ViRC 97. Il remplace l'événement par défaut OnChannelText. Il permet de visualiser une couleur par personne, ce qui augmente la lisibilité lorsqu'on est sur un channel assez fréquenté. De plus, une option "highlight" permet de beeper quand quelqu'un tape votre nick, ce qui vous avertit si vous êtes away.
La couleur de chaque personne est déterminée automatiquement par son user@host, toutefois il est possible de définir des couleurs personnalisées en ajoutant la personne dans la "user list" de virc.

Configuration requise

Visual IRC '97.

Installation

Decompressez le fichier dans votre répertoire de ViRC et tapez la commande /load BoDColored120.vsc dans n'importe quelle fenêtre (cette manipulation n'est nécessaire que la première fois).
Pour désactiver le "highlight" tapez /BoDColored h off. Pour désactiver la coloration tapez /BoDColored off (et biensûr remplacez "off" par "on" pour réactiver).
Pour ajouter une couleur personnalisée, allez dans la "user list" de virc (File / Client setup / IRC Settings / User lists), ajoutez ensuite le mask de la personne voulue. Les valeurs rouge, vert, bleu de la couleur voulue seront respectivement celles du Userlevel, Banlevel et Protlevel. Ces valeurs doivent etre entre 0 et 255, et le Userlevel doit etre différent de 0 pour que l'utilisateur soit pris en compte.

>>BoD Topic v1.02_

BoD Topic est un script pour ViRC 97. Il permet d'avoir une liste de topic prédéfinis qu'on peut appliquer facilement. Il est possible d'ajouter un suffixe par défaut (par exemple l'URL du channel). On peut également choisir aléatoirement un topic de la liste.

Configuration requise

Visual IRC '97.

Installation

Copiez le fichier dans votre répertoire de ViRC et tapez la commande /load BoDTopic.vsc dans n'importe quelle fenêtre (cette manipulation n'est nécessaire que la première fois).
Pour faire apparaître la fenêtre tapez /ttopic ou faites Ctrl + t. Vous pouvez aussi faire /ttopic suivi d'un topic. Cela appliquera le topic suivi du suffixe sur le channel en cours.

>>BoD Sex v0.26_

BoD Sex est une adaptation du logiciel du même nom de chez Nullsoft Inc. En gros, il fait exactement la même chose, mais en un peu mieux. Il s'agit d'un bloc-notes : dès qu'une idée vous passe par la tête, inutile de chercher un papier et un stylo, il suffit de cliquer sur le petit S en bas à droite.

  • une icone qu'elle est vachement mieux.

>>BoD Mr Random v1.02_

BoD Mr Random est un générateur de nombres aléatoires. Si vous avez une décision importante à prendre et que vous n'arrivez pas à vous décider entre plusieurs possibilités, une seule solution : s'en remettre au destin. Mr Random est fait pour vous. En un click il vous donnera la solution. Mr Random peut aussi servir à tirer beaucoup de nombres aléatoires en étant sûr qu'ils sont uniformément distribués (par exemple, pour un jeu de tests pertinant).



Particularités

Premier logiciel conforme à la norme BoD2000 : multiskin (possibilité de changer les icones :), en Java, et opensource (source fournie avec l'install).

Configuration requise

Plateforme acceptant le Java 2 (testé sous windows 98).

Installation

Pour utiliser Mr Random vous devez avoir une machine virtuelle Java dans votre système. Si vous n'en avez pas (ou si vous ne savez pas si vous en avez), allez sur java.sun.com et cherchez JRE (Java Runtime Environment), chargez-le et installez-le. Pour fonctionner, Mr Random nécessite au moins un environnement Java 2 (aussi appelé Java 1.2).
Vous pouvez ensuite décompresser le fichier d'install et lancer le fichier "Mr Random" si vous ètes sous windows. Si vous etes sous Unix ou autre, vous pouvez essayer de lancer le fichier mrrandom.bat.
